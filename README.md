# LLM Chat Application

An educational web application that demonstrates how to build a complete chat interface powered by a Large Language Model (LLM). This project uses the Hugging Face Inference API to interact with Meta's LLaMA 3.1-8B-Instruct model.

## üìö Project Purpose

This is a **learning-focused project** designed to teach the fundamentals of modern web development:

- Building a RESTful API with FastAPI
- Creating an interactive frontend with vanilla JavaScript
- Integrating third-party AI services (Hugging Face)
- Docker containerization
- Environment configuration and security best practices

---

## ‚ú® Features

- **Real-time chat interface** with a clean, user-friendly design
- **LLaMA 3.1-8B-Instruct** integration via Hugging Face
- **Type-safe API** using Pydantic models
- **Async/await architecture** for better performance
- **Comprehensive error handling** with user-friendly messages
- **Docker support** for easy deployment
- **Detailed logging** for debugging and learning

## üöÄ Installation & Setup

### 1. Clone the Repository

```bash
git clone <your-repository-url>
cd WebSandBoxProject
```

### 2. Set Up Python Virtual Environment

**Create virtual environment:**
```bash
python3 -m venv .venv
```

**Activate it:**

- **macOS/Linux:**
  ```bash
  source .venv/bin/activate
  ```

- **Windows:**
  ```bash
  .venv\Scripts\activate
  ```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure Environment Variables

**Create a `.env` file** in the project root:

```bash
cp .env.example .env
```

**Edit `.env`** and add your Hugging Face API key:

```env
HUGGING_FACE_API_KEY=hf_your_actual_key_here
```

> **Note:** You can get your API key from [Hugging Face Settings ‚Üí Access Tokens](https://huggingface.co/settings/tokens)

## üéØ Running the Application

### Option 1: Run Locally (Development)

**Make sure your virtual environment is activated**, then:

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**What this does:**
- `main:app` - Runs the `app` instance from `main.py`
- `--reload` - Auto-restarts the server when code changes (development only)
- `--host 0.0.0.0` - Makes the server accessible from other devices on your network
- `--port 8000` - Runs on port 8000

**Access the application:**
- Chat UI: [http://localhost:8000](http://localhost:8000)
- API Docs: [http://localhost:8000/docs](http://localhost:8000/docs) (auto-generated by FastAPI)
- Test Endpoint: [http://localhost:8000/test-api](http://localhost:8000/test-api)

### Option 2: Run with Docker

üìò **For a comprehensive Docker tutorial (in Russian), see [Docker_tutorial.MD](tutorials/Docker_tutorial.MD)**

**Build and start the container:**

```bash
docker-compose up --build
```

**Run in detached mode** (background):

```bash
docker-compose up -d
```

**Stop the container:**

```bash
docker-compose down
```

**Access the application:**
- Same URLs as above: [http://localhost:8000](http://localhost:8000)

## üîå API Endpoints

| Method | Endpoint      | Description                                    |
|--------|---------------|------------------------------------------------|
| GET    | `/`           | Serves the main chat interface (index.html)   |
| GET    | `/hello`      | Simple test endpoint (returns "Hello, World!")|
| GET    | `/test-api`   | Tests Hugging Face API connection             |

**Request:**
```json

Content-Type: application/json

{
  "message": "What is the capital of France?"
}
```

**Response:**
```json
{
  "response": "The capital of France is Paris."
}
```

## ‚ùó Troubleshooting

### Issue: "HUGGING_FACE_API_KEY not found"

**Solution:**
- Make sure you created a `.env` file in the project root
- Verify the API key is correct (starts with `hf_`)
- Restart the server after adding the key

### Issue: "‚è≥ The model is loading"

**Solution:**
- This is normal on first use or after inactivity
- Wait 20-60 seconds and try again
- The model goes to sleep when not used

### Issue: "Access denied. You may need to accept the model license"

**Solution:**
1. Visit [https://huggingface.co/meta-llama/Llama-3.1-8B-Instruct](https://huggingface.co/meta-llama/Llama-3.1-8B-Instruct)
2. Read and accept the license agreement
3. Wait a few minutes for access to be granted
4. Try again

### Issue: Port 8000 is already in use

**Solution:**
```bash
# Run on a different port
uvicorn main:app --reload --port 8001
```

Or find and kill the process using port 8000:

**macOS/Linux:**
```bash
lsof -ti:8000 | xargs kill -9
```

**Windows:**
```bash
netstat -ano | findstr :8000
taskkill /PID <process_id> /F
```

### Issue: Docker container won't start

**Solution:**
- Make sure Docker is running
- Check if `.env` file exists
- Rebuild the container: `docker-compose up --build`
- Check logs: `docker-compose logs`
